PROJECT: 4gag

Specificatii 4gag in mare

4gag o sa fie aplicatie dezvoltata in Laravel pe partea de back-end, folosind modelul arhitectural MVC ce comunica cu MySQL si cu clientul pentru a realiza stocarea, manipularea si transmiterea informatilor. Pe partea de front-end se foloseste React + tehnologii asociate si JQuery sau fetch() pentru request-uri asincrone.

Definitii

Post = entitate in pagina care are un titlu si o poza asociata, iar la apasarea pe poza, se deschide o pagina dedicata unde se pot posta comentarii si da "thumbs up/down". De asemenea, un post poate fi share-uita pe Facebook.

Aplicatia web o sa aiba urmatoarele features:

  • Login, Logout, Signup
  • Cand se intra pe site, orice utilizator o sa poata sa vada toate postarile, de la cele mai noi, la cele mai vechi
  • Posibilitatea de a face search dupa numele post-urilor
  • Posibilitatea de a gasi post-uri note dupa data adaugarii lor
  • Un design web responsive(poti sa folosesti un framework de genul Bootstrap/Foundation)
  • Post-urile se vor incarca in pagina asincron(cand se ajunge in jos-ul paginii, JS face un request AJAX, care o sa primeasca si o sa afiseze noile post-uri si tot asa
  • Orice utilizator logat o sa poata sa intre intr-un "backend" al site-ului si o sa poata adauga noi post-uri
  • Orice utilizator logat o sa poata salva postari intr-o categorie personala de "Favourites"
  • O sa existe clasa de SuperUser care are dreptul de a sterge postari considerate nepotrivite. Sesizarile unor postari malitioase pot fi facute de orice utilizator printr-un buton de "Report", iar un SuperUser poate viziona o lista cu astfel de reports si aproba stergerea sau chiar bann-area utilizatorului asociat.
  • Postarile vor fi organizate pe categorii

API

Aplicatia pune la dispozitia utilizatorilor inregistrati posibilitatea de a folosi un API ce are urmatoarele propietati:

  • Sa se poata obtine toate postarile create de utilizator
  • Sa se poata obtine toate comentarile postate de utilizatorul respectiv
  • Sa se poata obtine toate postarile salvate la "Favourites"
  • Sa se poata sterge o lista de titluri de postari ale utilizatorului.

Nota: sa se returneze un JSON cu un mesaj de eroare daca nu se poate ceva(specificandu-se motivul)
